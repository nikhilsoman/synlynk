synlynk v1.0 architecture brainstorm · 2026-06-03

Competency Is a Vector, Not a Role

The constraint on biological agents is cognitive load, not capability. The constraint on synthetic agents is context — size, depth, breadth, and retrieval speed.
Synlynk's rooms ARE the context-funding mechanism.

Biological Agent — Competency Ceiling

Specialisation is a cognitive load trade-off, not a hard limit. Full-stack exists but is rare and fragile under scale pressure.

Why specialisation happens: Cognitive load. Deep context in one domain displaces shallow context in others. 40-year careers converge, not expand.
Full-stack biological agents: Exist in early-stage orgs. Degrade under scale. Usually broad but shallow rather than deep across all axes.
Synthetic Agent — Competency as Context Function

No cognitive load constraint. The ceiling is context depth × breadth × retrieval speed. Full-stack is the default for a well-funded agent.

Context funding = room history depth + retrieval speed + cross-repo signal. A well-funded agent is full-stack by default.
Synlynk's job: be the context funding mechanism. Rooms accumulate depth. The agent's competency surface grows as the room history grows.
Competency Vector Schema — What synlynk tracks per agent-identity
Credential shape (competency as vector)

How competency scores are derived
  • PRs merged cleanly in each domain
  • Incidents caused (negative signal)
  • Room history depth in each subject area
  • Test pass rate on domain-specific tests
  • Peer review acceptance rate
Full-stack threshold

An agent is considered full-stack when all competency dimensions exceed the org-defined threshold (e.g., >0.75 across all axes). No special role needed — the vector speaks for itself.

Biological agents rarely sustain >0.75 across 6+ axes under production pressure. Well-funded synthetic agents do so by default once room history is deep enough.

Reframe: What Synlynk Actually Is

Synlynk is not a "context injector." It is a context funding mechanism — the infrastructure that accumulates room history, structures it, and delivers it to agents at retrieval speed. The deeper and broader the rooms, the more competent the agent. A well-funded synthetic agent achieves full-stack not because it's "smarter" — but because synlynk gave it enough structured history across all domains to act confidently in each.

This is the product story for Solo → Enterprise. Solo: fund your personal agents. Team: fund a shared competency pool. Enterprise: the org's accumulated context history IS its competitive moat — synthetic agents that know the org as deeply as any 20-year employee, on day one.
