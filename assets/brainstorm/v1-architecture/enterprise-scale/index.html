Does SQLite → Context Server Scale to Enterprise?

Honest answer: SQLite does not. But the room + message log abstraction does — if we pick the right substrate per tier.

The 3-Tier Hierarchy — What Each Layer Needs
Solo / Solo+
1 engineer · 1–4 repos · 1 machine
  • Local rooms (per branch)
  • WIP signal (no collision)
  • Agent handoffs
  • Context slices
  • No network needed
SQLite fits perfectly.
Single file, zero infra, Python stdlib.
Team (5–50 engineers)
Same timezone · shared GH org · 1–2 offices
  • Rooms replicated across machines
  • Real-time WIP across engineers
  • RBAC: who sees which rooms
  • Message history queryable by any member
  • LAN or single-region network
SQLite breaks here.
Single writer, no replication, no RBAC.
Enterprise (1000s)
Global · multi-org · strict compliance
  • Hierarchical rooms (team ⊂ org ⊂ enterprise)
  • RBAC with SSO/IdP integration
  • Audit log (immutable, retention policy)
  • Multi-region replication with low latency
  • Air-gap / self-hosted option
  • Global network, latency-sensitive
Needs purpose-built infra.

The Scaling Answer: NATS.io Leaf Node Architecture

NATS is purpose-built for exactly this pattern — and the local daemon we build for Solo+ maps perfectly to a NATS leaf node.

Why NATS fits this exactly
Subject = Room
rxcc.feat.di7 is a NATS subject. Wildcards: rxcc.> = all rxcc rooms. Hierarchy is native.
Leaf node = Solo daemon
The stdlib HTTP server we build for Solo+ becomes a NATS leaf node in Team mode. The API agents use doesn't change.
JetStream = durable history
Message log, replay, retention policies, consumers. The "room history" feature is JetStream consumers — same design, enterprise infra.
Message format across all tiers

NATS is payload-agnostic. We use MessagePack inside the payload at all tiers. At Solo it stays in SQLite BLOBs. At Team+ it rides NATS subjects as binary frames. No format change, no migration.

What this means for the build order

Design the room abstraction and message schema now, with NATS in mind. Build the Solo daemon against that abstraction using SQLite. When Team+ arrives, swap SQLite for a NATS leaf node — agents don't notice. The Solo layer is the NATS leaf node, minus the network hop.
