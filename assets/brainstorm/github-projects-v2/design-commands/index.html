Command Design — Step-by-Step Flows

What each command does internally

synlynk board connect <project-url>

  1. Parse org/repo + project number from URL
  2. GraphQL: fetch project metadata (id, fields, option IDs)
  3. Detect Agent field + Status field by name convention ("Agent", "Status")
  4. Write project_id + field schema to .synlynk/config.json
  5. Register repo in ~/.synlynk/config.json → registered_repos
  6. Print confirmation table: fields found, agent options, status columns
$ synlynk board connect https://github.com/orgs/Dialify/projects/1
✓ Project: Programme Board (PVT_kwDO...)
✓ Status field: 5 columns detected
✓ Agent field: claude, gemini, codex
✓ Registered in ~/.synlynk/config.json

synlynk start --issue <n>

  1. Fetch issue metadata (title, labels, current board item ID)
  2. Read WIP state from all registered repos (read path)
  3. Warn if issue's domain label routes to a different agent
  4. GraphQL: set Agent = self, Status = In Progress
  5. Add label agent:<name> to the issue
  6. Create git worktree at .synlynk/worktrees/<branch>
  7. Inject ## Live WIP block into .synlynk/context.md
  8. Print: "Ready. Working in worktree .synlynk/worktrees/fix-ocr-timeout"

synlynk pr [--title <t>]

  1. Read current issue from .synlynk/wip.json (set by start)
  2. GraphQL: set Status = In Review on board
  3. Derive suggested peer reviewer from review_rotation config
  4. Run gh pr create with Closes #N in body
  5. Assign suggested reviewer — print warning if overridden
  6. Post WIP comment on issue: "PR opened by <agent>"

synlynk done

  1. Verify PR is merged (warns if not)
  2. GraphQL: set Status = Done on board
  3. Archive checked-off tasks to devlog (existing checkpoint logic)
  4. Delete git worktree + local branch
  5. Clear .synlynk/wip.json
  6. Remove this issue from Live WIP in context.md

synlynk board status (read-only, no write)

Queries all registered boards, renders a terminal table of all In Progress + In Review items across repos. No daemon — one-shot query. This is the lightweight precursor to the v0.6 dashboard.

$ synlynk board status

REPO ISSUE AGENT BRANCH STATUS
rxcc #42 gemini feat/gemini/ui-timeline In Progress
rxcc #38 codex test/codex/auth-suite In Review
synlynk #12 claude fix/claude/context-slice In Progress
