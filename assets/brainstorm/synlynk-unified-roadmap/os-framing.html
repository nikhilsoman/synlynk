Synlynk as the OS for Multi-Agent Development

"OS" framing — what that means structurally, where competitors fit, and what it implies for the roadmap

The OS Layer Stack
📦

Applications

Third-party tools and skills that run on Synlynk — SuperPowers, GStack personas, custom skill packages, MCP servers.

Not Synlynk. These are ecosystem.
⌨️

Shell / Interface

The CLI the developer types into. synlynk run, dispatch, review, status, team. The TUI review interface.

synlynk run · synlynk review · synlynk status
🧭

Process Scheduler

Routes work to the right agent, manages the Architect→Build→Verify pipeline, tracks job state, retries failures.

trio.json · capability.json · job state machine
🔗

IPC — Inter-Agent Communication

Handoff context between agents, shared conventions layer, constraint propagation. How agents pass state to each other without talking directly.

conventions.md · handoff docs · constraint layer
🗄️

Filesystem — Persistent State

The durable, human-and-AI-readable project ledger. Survives session closes, tool switches, machine reboots. The only layer every agent shares.

project-docs/ · .synlynk/context.md · devlogs/
⚙️

Kernel — Process Lifecycle

Spawns and wraps agent CLI subprocesses, propagates exit codes, logs telemetry, enforces budget limits, Flatline Sentinel loop detection.

exec_command() · telemetry.json · flatline sentinel · Budget Pulse
Where Competitors Operate

SuperPowers

A skills library that runs inside a single agent session. Adds structured workflows and sub-agent task plans.

Lives at: Applications layer
Runs on the Shell. Has no Kernel, no FS, no IPC.

GStack

Persona framework for Claude Code. CEO→EM→QA pipeline with visual checks. A process scheduler with no persistent state.

Lives at: Shell + Scheduler
No FS, no IPC, no Kernel primitives. Claude-only.
The Key Insight

SuperPowers and GStack are applications. Synlynk is the OS they run on. You don't compete with them — you make them better. A SuperPowers skill running inside a Synlynk-managed session gets persistent memory, cost tracking, and multi-agent routing for free.

What the "OS" framing unlocks
🚫 No more tier model — OSes don't have Solo/Team/Enterprise editions. They have features that scale.
🤝 Complements competitors — SuperPowers users are the best Synlynk prospects.
📐 Roadmap = OS primitives — Kernel → FS → IPC → Scheduler → Shell → Ecosystem.
🌐 Platform play — v1.0 isn't "stable Lite." It's the stable API other tools build on.
