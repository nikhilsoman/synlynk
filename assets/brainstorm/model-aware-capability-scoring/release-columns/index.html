Features Page: Defining the Release Columns

How granular should the feature matrix columns be?

A

Group by minor version (v0.2.x, v0.3.x, v0.4.x…)

Each column = a minor release series. Patch releases collapse into one column. The matrix stays narrow even as releases accumulate. Simpler to read — users care about "when did X land" at the minor level.

Feature v0.1.x v0.2.x v0.3.x v0.4.x ↓
Context injection✓✓✓✓
Trio Protocol———✓
Board management———✓
B

Each version is its own column (v0.2.0, v0.2.1, v0.2.2…)

Every release gets a column. Shows exactly which patch introduced a feature. More precise but the matrix gets wide fast — at v1.0 you could have 15+ columns. Better for developers who care about exact version pinning.
