The root cause — state.db is the wrong sync unit

Syncing the full DB (export/import)

· Binary blob — can't diff, can't stream
· All-or-nothing — you get everything or nothing
· Merge is expensive — compare every table
· Manual cadence — humans forget
· No partial update — can't just send "here's the new constraint"
Syncing events (the right unit)

· Events are append-only — no conflicts possible
· Each event is tiny (200–500 bytes)
· Signed — cryptographically attributed
· Streamable — send only new events since last sync
· Materialized state = apply events in order

The fix — event-log sync via a shared channel

Architecture: local SQLite + event sync via shared git repo (pre-Tokq)
Each machine:
  state.db         ← local SQLite, fast queries, full schema
  events table    ← append-only log (already in our schema)

Sync channel (pre-Tokq):
  ~/.synlynk/workspaces/my-product/sync/  ← bare git repo
                                            OR shared GitHub repo
  nikhil/ → events.ndjson     ← Nikhil's new events (signed)
  gaurav/ → events.ndjson     ← Gaurav's new events (signed)
  kunal/ → events.ndjson     ← Kunal's new events (signed)

Daemon sync loop (every 5 min, or on every event write):
  1. Write new local events → sync/nikhil/events.ndjson → git push
  2. git pull → read gaurav/ and kunal/ events
  3. Apply new events to local state.db (idempotent, ordered by ts)
     → story status updated · memory updated · constraints visible

Result: max drift = ~5 minutes. Constraint added at 09:14 visible to Gaurav by 09:19.

Path to Tokq Alpha — same events, different transport

Pre-Tokq (v0.6.0+)

Events → git repo sync channel
Daemon polls every 5 min
Per-member branches, no conflicts
Shared GitHub repo = team hub
Max drift: ~5 minutes
Tokq Alpha (NATS)

Events → NATS subjects (same format)
Daemon subscribes, real-time push
ZK-encrypted in transit
Tokq hub = team hub
Max drift: <1 second
The event format is identical in both cases. The only thing that changes is the transport layer — git repo → NATS subject. The daemon sync loop is the same code with a different I/O backend. Migration cost at Tokq Alpha: near zero.
