Proposed page structure — synlynk.com

Scrolling single-page landing + /blog/ for posts. Click any section to flag it for discussion.

1

Hero — dark full-width

Logo lockup (icon + wordmark), one-line tagline, dark terminal background. Install one-liner in a copyable code block. Two CTAs: Get started → and View on GitHub. Optional: animated typing effect showing synlynk exec claude.

2

The problem — light background

2–3 short sentences: every AI session starts from zero, context is lost between tools, costs are invisible. No bullet points — tight prose. Possibly with a simple "before/after" diagram: Claude forgets → synlynk remembers.

3

How it works — light background

Numbered 4-step flow: init (bootstrap docs) → exec (context injected) → checkpoint (archive done tasks) → status (see budget + alerts). Each step has a small terminal snippet and one-line description. Uses logo accent colors for step numbers.

4

Features — light background, 4-card grid

Four cards in a 2×2 grid:
⟡ Context injection — shared project-docs/ every AI tool reads
⟡ Cost tracking — budget limits, 80% warnings
⟡ Watch daemon — context.md stays fresh as you work
⟡ Sentinel alerts — detects flatline failure loops
Each card uses an SVG icon or Unicode glyph, title, and 1-sentence description.

5

Install — subtle dark band

One install line: curl -sSL … | bash. Secondary: python3 bin/synlynk.py for no-install use. Requirements badge: Python 3.8+, stdlib only. GitHub stars badge. Keeps it honest and minimal.

6

Roadmap — light background

Three-stage horizontal progression with status indicators:
● Solo edition (current — v0.2.0)
○ Team edition (next)
○ Enterprise edition (future)
Short paragraph per stage. Links to GitHub Discussions for community input.

7

Blog — light background

Section header "From the build log" or "Dev notes". Lists 3 most-recent posts with title, date, and one-line excerpt. See all posts → links to /blog/. Individual posts live at /blog/slug/ and use a focused reading layout: no sidebar, max-width prose, syntax highlighting.

8

Footer — dark to match hero

Small logo mark, GitHub link, Discussions link, MIT license note, current version (v0.2.0). One line. No newsletter, no social links beyond GitHub — honest about where the project is today.

Blog posts (launch set):
📝 "Why I built synlynk" — the original problem: context loss across AI tools
📝 "From idea to v0.1.0: the first working CLI" — what it was, what it wasn't
📝 "Redesigning for v0.2.0: watch daemon, checkpoint, and context compaction" — the iteration, why
📝 "Release notes deep dive: v0.2.0" — detailed changelog walkthrough
