Building flatline four ways — the controlled experiment that proves the claim

The Target: flatline

A standalone Python CLI circuit-breaker. flatline pnpm test wraps any shell command, hashes stdout on failure, and kills the process after 3 consecutive identical failures — saving developers from runaway agent token bills.

~150 lines Python stdlib only 3-session build pip-installable
S1: design CLI + spec
S2: implement hashing logic
S3: tests + package

The 4 Rounds

R1 Bare Agent

Claude Code with no skill packs, no synlynk. 3 sessions. Baseline token spend + coverage.

Environment: fresh Claude Code install, empty project
R2 Superpowers only

Claude Code + Superpowers skill pack. Same 3-session task. Measures skill pack overhead and quality lift.

No synlynk. Superpowers TDD + brainstorming skills active.
R3 synlynk only

Claude Code + synlynk, no skill packs. Measures synlynk's context injection and cost tracking contribution in isolation.

project-docs/ initialized. Context injection active. Flatline sentinel on.
R4 synlynk + Superpowers

The "better together" round. synlynk provides state + cost control; Superpowers provides workflow discipline. Expect lowest token spend + highest coverage.

git-drift pre-commit hook active — no instruction conflict possible.
