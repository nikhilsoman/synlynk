Section 2 — Coexistence Theory (AB-11)

How synlynk:start/end blocks, Superpowers hooks, and GStack additions share instruction space without conflict

The Instruction Surface Problem

synlynk writes to
  • CLAUDE.md (OS preamble)
  • GEMINI.md
  • AI_INSTRUCTIONS.md
  • .synlynk/context.md
  • .cursorrules
Superpowers writes to
  • CLAUDE.md (skill hooks)
  • settings.json
  • keybindings.json
  • slash command hooks
GStack writes to
  • CLAUDE.md (quality rules)
  • .cursorrules
  • custom lint hooks
Overlap zone: CLAUDE.md and .cursorrules — all three write here. Without a priority contract, agents see contradictions.

Conflict Taxonomy (AB-11)

Conflict Type Example Symptom Resolution
Direct contradiction synlynk: "always run tests before commit" · GStack: "never block on tests in draft PRs" Agent skips tests randomly or asks for clarification every time synlynk block = authoritative; skill pack rule = advisory
Scope bleed Superpowers TDD skill injected into a `synlynk exec agy` session not designed for TDD Token bloat; agent follows TDD steps for a 2-line config change Context scoping: only inject skill hooks matching current story's role
Format collision GStack adds its own ## Code Quality section; synlynk already has one Duplicate sections confuse agent; one is silently ignored git-drift flags duplicate headings at commit time
Authority ambiguity Both synlynk and Superpowers claim "you MUST follow these rules" Agent paralysis or arbitrary priority selection synlynk:start/end fencing declares OS-level authority explicitly

The Resolution Protocol

synlynk:start/end fencing

# OS Layer Instructions
[persistent memory, budget limits,
 dispatch rules, flatline policy]


# Skill Pack Zone
[Superpowers / GStack rules go here
 — advisory, not authoritative]

OS-level rules are fenced and treated as non-overridable by agents. Skill pack rules below the fence are advisory.

git-drift enforcement
  • Pre-commit hook scans CLAUDE.md
  • Detects missing synlynk:start/end fence
  • Flags duplicate section headings
  • Warns if skill pack rules above the fence
  • Compares against team manifest baseline

Runs on every commit — no drift accumulates silently over time.
