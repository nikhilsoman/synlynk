Design — Section 3: Entitlements

Authorization (what can start) + Sandboxing (what it can do while running)

Two entitlement layers

Authorization — gate before dispatch
Answers: can this story auto-start?
Checked by daemon before dispatch.

Dimensions:
· cost ceiling — auto if estimate < $X
· path sensitivity — infra/, auth/, main branch
· action type — create PR vs merge to main
· novelty — first run of this domain needs approval
· competency floor — agent score below threshold → hold

Outcome: AUTO-DISPATCH · HOLD-FOR-APPROVAL · REJECT
Sandboxing — constraints while running
Answers: what can the agent touch?
Enforced by synlynk wrapper around exec.

Dimensions:
· filesystem paths — read/write per glob pattern
· git ops — branch · commit · push · PR · merge
· network — external API calls allowed/blocked
· token ceiling — max tokens per story execution
· time ceiling — max duration before check-in required

Outcome: ALLOW · BLOCK · ESCALATE-TO-HUMAN

Auto-dispatch if cost < $0.50 ✓ ✓ ✓

✓ auto = dispatches without approval  ·  ⚠ approval = email sent, waits for reply  ·  ✗ = blocked, never dispatched in this role

Approval flow — email bridge in action

Story requires approval → email → human replies → dispatch or hold
story #42: "migrate auth to OAuth2" · role=Builder · domain=auth · estimate=$1.20
  → cost > $0.50 threshold AND domain=auth (sensitive) → HOLD-FOR-APPROVAL
  → emit event type="approval_requested" · story_id=42
  → email to nikhilsoman@gmail.com:
     "Story #42 ready · Builder/claude-code · est. $1.20 · touches src/auth/
      Reply YES to dispatch · HOLD to defer · REASSIGN claude→gemini"

human replies "YES"
  → synlynk reads reply (Gmail API or webhook)
  → emit event type="approval_granted" · user="nikhilsoman"
  → daemon dispatches story #42 to Builder/claude-code
  → entitlement record: approved by nikhilsoman at 14:32

Where entitlements live in state.db

entitlements table — one row per (role, dimension, value). Defaults seeded at synlynk init. Editable via synlynk entitle Builder write:src/ allow.
Per-story overrides: a story can declare required_approval: true regardless of role thresholds.
All entitlement checks and outcomes written to events table — full audit trail.
