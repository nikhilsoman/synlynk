synlynk HUD — terminal presence

When does it appear, what does it show, where does it live?

A — Bottom bar (always visible, ambient)

Persistent strip at bottom of terminal. Shows live state at a glance. Like tmux status or starship prompt — always there, low noise.

synlynk — ~/dev/synlynk
$ synlynk exec claude
✓ Context saved to .synlynk/context.md
✓ Dispatching claude...
synlynk
|
story auth-oauth-flow
|
agent claude ● running
|
budget $3.40 / $10
|
relay 2 peers
|
alerts none

B — Contextual pop-up (appears on events, auto-dismisses)

HUD is invisible until something needs attention. Pops in from top or bottom when: job completes, budget threshold crossed, sentinel fires, relay disconnects. Dismisses after 5s or on keypress.

~/dev/synlynk
✓
job-34ab991e completed
codex · fix bare open() in cmd_identity_init · 8s · 365 tests pass
↵ dismiss · s synlynk logs
$ git commit -m "feat: add OAuth flow"
[main 4f2c1a3] feat: add OAuth flow
$ █
⚠
Budget 80% used — $8.12 of $10.00
relay · nikhil synced 3 stories from teammate AGY run
↵ dismiss
$ synlynk dispatch agy --story auth-001 --task "write tests"
✓ job-a1b2c3d4 started

C — Hybrid: thin ambient bar + rich pop-up on events

A minimal 1-line bar is always present (story + budget + relay health). When an event fires, it expands into a richer notification in-place. Collapses back automatically. Like a macOS menu bar that pops open briefly.

$ git push origin feat/auth

$ git push origin feat/auth

● auth-oauth-flow ·$3.40 ·relay 2↑ ·no alerts
A

Bottom bar — always visible

Ambient. Low-noise. Always there like a tmux statusline. Good for solo power users who want constant situational awareness.

B

Contextual pop-up only

Invisible until something happens. Zero distraction baseline. Respects terminal real estate. Better for newcomers and pair sessions.

C

Hybrid: thin ambient bar + expands on events

Minimal presence always. Rich notification on demand. Collapses back. Best of both — awareness without noise.
