Agentic PM — Full Hierarchy with Three Evolution Patterns

Derived from synlynk (pivot-heavy), rxcc (scale-complex), cc-videoreframing (dormant/resumable)

The three evolution archetypes observed

synlynk / Tokq

Pivot-heavy

Context injector → multi-agent fabric → OS layer. Tokq was a separate cloud project, then converged. Strategic arcs pivot, merge, and supersede. The archived proposals folder is the history of what we thought before we knew better.

rxcc

Scale-complex

Linear direction but global scope with local nuance. No strategic pivots — but constant reprioritization within a stable direction. WhatsApp, vocab pipeline, GA4 all move up and down the queue. The complexity is in sequencing parallel tracks.

cc-videoreframing

Dormant / resumable

Script → AWS infra → SaaS plan → paused. MEMORY.md has a full chronological history specifically because re-entry requires: current state, last decisions, what's blocked, what comes next. The SaaS plan sits waiting.

Full agentic hierarchy — revised with Arc layer

Project → Arc → Phase → Epic → Story → Event
Arc      "OS for multi-agent dev"  ← NEW LAYER — the missing piece
↳ a strategic direction. not time-bound, intent-bound.
↳ status: active | archived | merged
↳ pivot = archive old arc + start new arc + record external_trigger
↳ convergence = two arcs merge, stories migrate to merged arc
↳ human alias: "Phase 1", "v1 strategy" — optional

Phase    "SQLite Kernel"  ← capability gate within an arc
↳ has: capability_assertion, depends_on_phase, priority_order
↳ reprioritize = change priority_order + emit reprioritize event
↳ human alias: "v0.5.0", "Aug 2026" — optional labels only

Epic     "Context Bridge"  ← agent-domain cluster (parallelism surface)
↳ defines what can run concurrently within a phase
↳ routable: assignable to a specific agent or human

Story    "migrate project-docs to state.db"  ← was: task
↳ done_criteria: "synlynk exec reads from state.db, not files"
↳ depends_on: [story_id_1, story_id_2] — graph, not list
↳ no time estimate. done when criteria pass, not when time elapses.

Event     ← the universal log. replaces devlog. covers everything.
↳ types: started · completed · blocked · decision · note
↳ pivot · reprioritize · checkpoint · resume · external_signal
↳ devlog = SELECT * FROM events WHERE user='nikhil' ORDER BY ts DESC

Why Arc is the missing piece in every PM tool

Jira, Linear, GitHub Projects all treat strategic direction as implicit context — a README, a vision doc, a Notion page. When you pivot, you create a new project and the history of why you thought the old direction was right is severed. Arc makes strategic direction a first-class entity in the data model. Pivots become traceable. Convergences become modelable. The archived proposals folder becomes a queryable table.
