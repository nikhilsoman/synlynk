Final Schema — Corrected & Verified

All tables with clarified roles. Memory stays. Arc is new. Phase/Epic/Story replace roadmap/tasks.

Hierarchy — roles locked

Project → Arc → Phase → Epic → Story → Event
Arc     Strategic direction. Pivots, convergences, external triggers. Active | Archived | Merged.
Phase   Structural project backbone. Rarely changes. The "bones."  ← was: roadmap row
Epic    One implementation plan.  ← what superpowers:writing-plans generates
Story   One agent task unit. Assigned, verifiable.  ← was: task row
Event   State transition log. Append-only.  ← replaces devlog
Memory Curated durable decisions. Stays as-is. decision events may overlap — expected.

arcs

id INTEGER PK
name TEXT          ← "OS for multi-agent dev"
status TEXT        ← active | archived | merged
rationale TEXT     ← why this arc exists / why archived
external_trigger TEXT ← what caused a pivot
merged_into INTEGER ← FK to arc.id if merged
created_at TIMESTAMP
archived_at TIMESTAMP

synlynk example:
"multi-agent fabric" → archived
"OS layer" → active
"Tokq cloud" → merged into "OS layer"

phases

id INTEGER PK
arc_id INTEGER FK
name TEXT          ← "SQLite Kernel"
os_layer TEXT      ← "Scheduler — data-driven routing"
infrastructure TEXT  ← "SQLite WAL"
capability_assertion TEXT ← "after this, X works"
status TEXT        ← shipped | next | planned
priority_order INTEGER ← reorderable, history in events
target_alias TEXT   ← "v0.5.0 · Aug 2026" (human label)

Structural backbone. Rarely changes.
Reorder via priority_order + reprioritize event.

epics

id INTEGER PK
phase_id INTEGER FK
name TEXT    ← "Context Bridge"
source TEXT   ← "superpowers-plan" | "manual"
owner TEXT    ← agent or user assigned
status TEXT   ← pending | active | done
created_at TIMESTAMP

One implementation plan = one epic.
writing-plans creates the epic + all its stories in one shot.
The parallelism surface within a phase.

stories

id INTEGER PK
epic_id INTEGER FK
text TEXT          ← "migrate project-docs to state.db"
done_criteria TEXT  ← "synlynk exec reads from state.db"
depends_on TEXT     ← JSON [story_id, ...] — graph not list
status TEXT        ← pending | in_progress | done | blocked
owner TEXT         ← agent name
created_at TIMESTAMP

Atomic agent task unit. Done when done_criteria pass,
not when time elapses. No story points.

events

id INTEGER PK AUTOINCREMENT
type TEXT       ← started | completed | blocked
                     decision | note | external_signal
                     pivot | reprioritize | checkpoint | resume
story_id INTEGER  ← nullable FK
epic_id INTEGER   ← nullable FK
phase_id INTEGER  ← nullable FK
arc_id INTEGER    ← nullable FK
user TEXT
content TEXT      ← narrative or structured JSON
ts TIMESTAMP

Universal append-only log.
devlog = SELECT WHERE user='nikhil' ORDER BY ts DESC
pivot history = SELECT WHERE type='pivot'

memory + costs — unchanged

memory: heading | body | agent | updated_at
← curated durable decisions, full sections
← decision events may overlap — expected

costs: session_date | user | model | tokens_in
       | tokens_out | cost_usd | ts
← session ledger, not injected into context

superpowers integration point

writing-plans output → one Epic row + N Story rows inserted into state.db under the current Phase. The plan document becomes the source of record for the Epic; the individual tasks become Stories. synlynk run picks up pending Stories from the active Epic and dispatches them to agents.
