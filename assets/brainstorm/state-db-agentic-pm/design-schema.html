state.db table schemas and the one-time migration from project-docs/

state.db — table schemas

agent    TEXT        ← who wrote it

id       INTEGER PK

owner    TEXT        ← agent name

devlog

id      INTEGER PK AUTOINCREMENT
session TEXT   ← date + seq
agent   TEXT
content TEXT
ts      TIMESTAMP
append-only in practice

costs

agent       TEXT
model       TEXT
tokens_in   INTEGER

roadmap

key   TEXT PRIMARY KEY
value TEXT

← mode, version, project_name,
   init_ts, limit_usd, limit_requests

Migration — one-time, synlynk migrate

What migrate does
1. Create ~/.synlynk/projects/<project_id>/state.db
2. Parse project-docs/memory.md  → INSERT INTO memory
3. Parse project-docs/todo.md    → INSERT INTO tasks
4. Parse project-docs/costs.md   → INSERT INTO costs
5. Parse project-docs/devlogs/*  → INSERT INTO devlog
6. Parse project-docs/roadmap.md  → INSERT INTO roadmap
7. Add project-docs/ to .gitignore
8. git rm --cached project-docs/  ← untrack without deleting
9. Print summary + prompt to commit .gitignore change

Files remain on disk after migration — nothing deleted. Agents just stop seeing them in git and stop writing them as files.
