❌ Old mapping (broken after v0.5.0)

"One memory unit per project-docs/*.md file" — references files that no longer exist after synlynk migrate
Tokq fieldOld synlynk source
memory_id sha256(session_id + filename + version)

encrypted_data Full file content of project-docs/*.md
— file doesn't exist after v0.5.0 migrate

metadata.tags [@username] scraped from file text

⚠ After synlynk migrate (v0.5.0), project-docs/ is gitignored. The file-grain mapping silently breaks. A sync run after migration would produce empty or stale memory units.

✓ New mapping (state.db view-grain, v1.0)

One memory unit per purpose-typed state.db view. Structured JSON payload, not raw file content. Backward-compat: file-grain used if state_db: false in config (pre-migration).
Tokq fieldNew synlynk source
memory_id sha256(workspace_id + unit_type + entity_id + version)
entity_id = phase.id | epic.id | workspace.id | date_bucket
encrypted_data Structured JSON view of state.db tables
— generated fresh, never stale
metadata.title "<unit_type>:<entity.title>"
metadata.tags DB-sourced tags (arc.tags, epic.labels, event.type set)
granularity semantic-grain — scales with project: N epics → N execution units
✓ Idempotent: same entity_id + same state → same memory_id. ✓ Scales: a project with 20 epics → 20 execution units, 1 context unit, N strategic units. ✓ Queryable: Tokq can reconstruct a project template from strategic units alone.

The 5 Memory Unit Types

strategic
publishable
Arc + Phase roadmap structure
source: arc + phase tables
replaces: roadmap.md
entity_id: phase.id
payload: {arc, phases[], generated_at}
tags: arc.tags + phase.labels
→ synlynk publish conventions packages these as a Tokq collection. Subscribers get your project structure as a template.
context
opt-in publish
Architecture decisions + positioning
source: memory table rows
replaces: memory.md
entity_id: workspace.id (one per workspace)
payload: {decisions[], architecture[], positioning}
tags: domain tags from memory table
→ Optionally published as "architecture patterns" collection. Private by default — team shares via workspace sync.
execution
private
Epic + open stories (current work)
source: epic + story (status='open')
replaces: todo.md + partial memory.md
entity_id: epic.id (N units per project)
payload: {epic, stories[], done_criteria[]}
tags: epic.labels + story.labels
→ Scales with work: 1 epic → 1 unit. Cross-machine team sync delivers active task state without sharing raw code.
activity
private
Per-user event stream (replaces devlogs)
source: event table, git_user + time window
replaces: devlogs/<user>.md
entity_id: git_user + ISO_date_bucket (daily)
payload: {user, events[], date_range}
tags: git_user + event.type distinct values
→ Always private. Team members sync their own activity units; teammates see them via shared workspace event stream.
capability
opt-in sync
Agent fitness scores (no project-docs equivalent)
source: agent_profiles + scoring history
replaces: (new — nothing in project-docs)
entity_id: agent_uuid + scoring_period
payload: {agent_profiles[], scores[], half_life_decay}
tags: agent.name + domain + phase.type
→ Machine-local by default. Opt-in to sync enables "bring your capability profile to a new machine / new project" use case.
never synced
costs.md / costs table local ops ledger — separate from Tokq gas tank (ledger boundary, FR-6)

external_refs workspace-specific GitHub/Jira/Linear links — not portable

sentinel.md ephemeral alert state — no historical value

Updated memory_id Formula

old
sha256(session_id + filename + version_counter)
filename = "memory.md" | "roadmap.md" | …
→
new
sha256(workspace_id + unit_type + entity_id + version_counter)
entity_id = phase.id (strategic) | workspace.id (context) | epic.id (execution) | git_user+date (activity) | agent_uuid+period (capability)
workspace_id replaces session_id — stable across multi-repo workspaces (not per-repo)
