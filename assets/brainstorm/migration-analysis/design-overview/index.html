Feature 1 — synlynk migrate

Flow
1. Scan — find project docs (roadmap.md, todo.md, memory.md, costs.md, devlogs/) anywhere in repo tree
2. Detect repo maturity — agent file >100 lines OR 3+ unknown sections → "evolved repo" path
3. Evolved repo menu:
   A — Adopt + combine: semantic section match, extract GH IDs, add only missing sections
   B — Exit (no changes)
   C — Full replace (requires typing 'replace')
4. Fresh repo: normal init flow, move docs to project-docs/
5. Always: extract GH Projects v2 IDs into .synlynk/config.json if found
6. Always: create AGENTS.md if missing
7. synlynk init nudge: "Existing docs detected outside project-docs/ — run synlynk migrate"

Feature 2 — synlynk start <issue-id>

Flow
1. Read issue — gh api repos/:owner/:repo/issues/:id → body + labels
2. Infer agent — find agent:<name> label → map to exec command
3. Discover board fields — check .synlynk/board-cache.json; if stale/missing, query GH Projects v2 GraphQL → cache result
4. Move board — set Status = In Progress + Agent field via GraphQL
5. Build context — generate_context() + prepend issue body + domain label
6. Launch agent — exec_command([agent_exec]) with context injected
