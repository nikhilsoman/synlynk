Agent discovery

init wizard scans for Claude, Gemini, Codex, and AGY — shows your workgroup at a glance

What it does

`synlynk init` now runs a 6-step wizard. Step 2 is Magic Moment 1: it probes each known agent CLI with `--version`, then prints a workgroup table showing every installed agent with its roles (Architect / Builder / Verifier) and a functional indicator. `discover_agents()` reads `AGENT_CAPABILITY_BASELINES` for role and flag definitions, checks `~/.claude`, `~/.gemini`, etc. for config directories, and supports per-project path overrides in `.synlynk/config.json`. An opt-in LLM enrichment step calls the best available agent non-interactively to synthesise an informed `roadmap.md` from git history and the README.
