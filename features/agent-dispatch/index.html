Background dispatch

Fire-and-forget agent jobs from any shell — synlynk dispatch claude --task "..."

What it does

`synlynk dispatch <agent> --task <text>` launches the agent CLI in the background using `subprocess.Popen(start_new_session=True)`, capturing stdout to `.synlynk/logs/<job_id>.log` and persisting the PID to `.synlynk/jobs.json`. On every startup, `_reconcile_jobs()` probes live PIDs and marks dead ones as completed or failed. `synlynk jobs` lists running/recent jobs; `synlynk logs --job <id>` tails output in real time.

Usage

$ synlynk dispatch claude --task "refactor auth module" Launches claude in background, returns job ID immediately
$ synlynk jobs Lists all running and recent jobs with status, agent, and task
$ synlynk logs --job job-abc123 --tail 50 Tails the last 50 lines of a job's stdout log
