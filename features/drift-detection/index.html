Instruction drift detection

Fires INSTRUCTION_DRIFT sentinel when a tracked instruction file is edited outside synlynk

What it does

Every `synlynk exec` call runs `_check_instruction_drift()`, which loads `.synlynk/instructions.json` (the SHA manifest written by `init`) and compares the SHA of each file's synlynk-owned section against the stored value. If any file has been edited, an `INSTRUCTION_DRIFT` alert is written to `sentinel.md`. The sentinel fires once per change — not once per exec — by updating the manifest SHA immediately after firing. Use `synlynk instructions update <file>` to regenerate the section, or `synlynk instructions ack <file>` to dismiss the alert without regenerating.

OS Layer

Kernel — Context Engine

Introduced in

Usage

$ synlynk instructions diff Shows user content outside synlynk markers in each tracked file
$ synlynk instructions update CLAUDE.md Regenerates the synlynk section and resets its SHA in the manifest
$ synlynk instructions ack CLAUDE.md Removes INSTRUCTION_DRIFT lines from sentinel.md without regenerating
