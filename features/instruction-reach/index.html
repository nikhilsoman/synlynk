Instruction reach

Extends synlynk context injection to Cursor, GitHub Copilot, and Windsurf

What it does

v0.4.1 extends `synlynk init` from 3 CLI targets to 7 tracked instruction files. In addition to `CLAUDE.md`, `GEMINI.md`, and `AGENTS.md`, synlynk now writes tool-native instruction blocks into `.cursor/rules/synlynk.mdc` (Cursor, with `alwaysApply: true` MDC frontmatter), `.github/copilot-instructions.md` (GitHub Copilot), `.windsurfrules` (Windsurf), and `AI_INSTRUCTIONS.md` (universal fallback). Three marker styles are used to preserve user content: `html` comments, `hash` (`# synlynk:start`), and `none` (synlynk owns the whole file). Cursor and Copilot targets are only written if their respective config directories exist.

Usage

$ synlynk init Writes synlynk section to all 7 instruction targets; creates .cursor/rules/synlynk.mdc if .cursor/ exists
$ synlynk instructions status Shows status (ok / missing / drifted / untracked / no-markers) for each tracked file
