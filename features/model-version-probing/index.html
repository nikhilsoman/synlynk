Model version probing

Records which model version ran each job — enabling split-model detection and honest routing

What it does

v0.6.0 adds four-tier model version resolution. At dispatch time, `_probe_model_version()` fires a `--version`-style call to the agent CLI and scrapes the model identifier (e.g. `claude-opus-4-8`). This becomes `model_at_dispatch` on the job record. A `# synlynk-meta` block in the agent's output can supply `model_at_completion` (Tier 1, highest trust). If both are known and differ, `split_model=1` is set, excluding the run from capability aggregation. `synlynk score attest` allows retroactive attestation when the agent reports its model post-run.

Usage

$ synlynk score attest --job job-abc123 --model claude-opus-4-8 Retroactively records which model completed the job
$ synlynk pr check Blocks merge if any job in the PR used an unknown or mismatched model version
