Context injection

Reads project-docs/ into .synlynk/context.md before each exec

What it does

Before every `synlynk exec` call, the harness reads `project-docs/memory.md`, `roadmap.md`, and `todo.md` into a single `.synlynk/context.md` file that is prepended to the agent's prompt. This ensures every AI session starts with current project state — no repeated setup, no lost context.

OS Layer

Kernel — Context Engine

Introduced in

v0.1.x — Proof of Concept

Usage

$ synlynk exec claude Generates .synlynk/context.md then spawns claude with context prepended
