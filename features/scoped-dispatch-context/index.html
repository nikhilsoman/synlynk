Scoped dispatch context

Task-level context slice replaces 7-day devlog dump in every agent dispatch

What it does

v0.9.0 introduces `generate_context(scope="task:<story_id>")`. Instead of injecting the full project context (7-day devlog, all teammate activity, complete memory file) into every dispatch, `_generate_task_context()` builds a minimal snapshot: story metadata from state.db, active tasks only from todo.md, and up to 20 source files filtered by the story's engineering domain from the scan cache. This reduces token overhead per dispatch by 60–80% and keeps agents focused on the relevant subsystem.
