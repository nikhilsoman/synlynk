Telemetry log

Rolling 100-entry log of every exec — duration, exit code, cost, command

What it does

Every `synlynk exec` invocation appends a structured JSON record to `.synlynk/telemetry.json`, keeping only the last 100 entries. Each record includes: timestamp, command, duration_seconds, exit_code, input_tokens, output_tokens, and estimated_cost_usd. This log powers both flatline detection and budget enforcement.

Filesystem — State & Telemetry

Introduced in

v0.1.x — Proof of Concept

Usage

$ cat .synlynk/telemetry.json | python3 -m json.tool | tail -30 Inspect last 100 exec records
