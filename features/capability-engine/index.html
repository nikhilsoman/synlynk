Capability engine

Routes tasks to the best agent based on a live SQLite ledger of quality scores

What it does

v0.5.0 introduces data-driven agent routing. `_best_agent_for_story(story_id)` queries the `capability_scores` SQLite view, which aggregates human ratings, verifier pass rates, and auto-quality signals across a 3D domain taxonomy (engineering domain × org domain × industry). A cold-start round-robin runs for the first 3 samples per (agent, domain) triple; recency-weighted scoring takes over after that. The `synlynk story create/list` and `synlynk score add/list` commands provide the CLI surface for the capability ledger.

Usage

$ synlynk story create "Implement OAuth flow" --domain auth Creates a story record in state.db, returns a story_id
$ synlynk score add --story story-abc123 --agent claude --quality 4 Records a human quality rating (1–5) for a completed run
$ synlynk score list Shows aggregated capability scores per agent per domain
