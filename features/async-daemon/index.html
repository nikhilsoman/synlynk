Async daemon

Always-on process — agents pull context over HTTP without invoking the CLI

What it does

`synlynk daemon start` double-forks a background process that keeps `context.md` current via mtime polling and exposes it over a local HTTP API on `localhost:27471`. A persistent `daemon_jobs` queue in `state.db` tracks jobs with priority ordering, dependency chains, and crash-safe per-job commits. `--install-service` registers the daemon with launchd (macOS), systemd user units (Linux), or a `@reboot` crontab entry as a fallback — so it survives reboots without manual intervention.

Usage

$ synlynk daemon start Fork-daemonize; writes .synlynk/daemon.pid; HTTP server starts on localhost:27471
$ synlynk daemon status Shows pid, uptime, queue depth (queued/running/done/failed), and HTTP URL
$ synlynk daemon --install-service Registers with launchd (macOS) / systemd (Linux) / crontab (fallback) for auto-start on login
$ curl http://localhost:27471/context Agents fetch current context.md without invoking synlynk exec
$ curl -X POST http://localhost:27471/dispatch -d '{"agent":"claude","task":"...","priority":3}' Enqueue a job; daemon picks it up on the next poll tick
