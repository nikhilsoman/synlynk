Flatline detection

Alerts and halts when the last 3 runs share identical command + non-zero exit

What it does

synlynk tracks the last 100 exec invocations in `.synlynk/telemetry.json`. If the most recent 3 entries share the same command and all exited non-zero, the harness prints a flatline warning and blocks the next run. This prevents silent looping where an agent repeatedly fails at the same step without human review.
