Ed25519 capability signing

Every capability rating is signed with a local Ed25519 key — tamper-evident from day one

What it does

v0.9.0 wires the `ed25519_sig` column in the `capability_ratings` table that has existed since v0.5.0. `_ensure_identity_key()` creates `~/.synlynk/identity.key` via `ssh-keygen -t ed25519` on first use. `_sign_capability_rating()` signs a canonical `json.dumps(..., sort_keys=True)` payload of the rating fields via `ssh-keygen -Y sign`, then stores the signature alongside the rating. This is the first step toward the Tokq-compatible signed capability ledger planned for v1.0.0.
