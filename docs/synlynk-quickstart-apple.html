Chapter 01 — Overview 02
What is Synlynk?

Your AI agents,
working together.

Synlynk is a single-file Python CLI that wraps AI agents — Claude, AGY, Codex — and makes them project-aware, cost-tracked, and safe to run in parallel. One file. Zero dependencies. Works anywhere Python 3 runs.

Why Synlynk?

🧠
Context-Aware by Default
Every agent invocation automatically receives your roadmap, memory, and todos. No more copy-pasting context into every session.
💰
Zero Surprise Bills
Budget guard blocks execution the moment you hit your daily or session limit. Real-time cost tracking on every exec.
🛡️
Hallucination Sentinel
Automatically detects flatline loops, success loops, and quota exhaustion. Alerts before damage is done.
⚡
Parallel Dispatch
Launch Claude, Gemini, and Codex simultaneously on different tasks. Monitor all jobs from one CLI while your terminal stays free.
👥
Team-Ready
Single or team mode. Per-user devlogs with git attribution. Shared roadmap and memory visible to every agent in the repo.
🔌
Zero Dependencies
Pure Python 3 stdlib. No pip install, no Docker, no build step. One file — works in any environment where Python 3 runs.

Chapter 02 — Installation 03
Getting Started

Up and running
in 60 seconds.

1
Clone & install globally
Adds synlynk to your PATH via ~/.synlynk/bin/. No sudo, no pip, no build step required.
install
$ git clone https://github.com/nikhilsoman/synlynk
$ cd synlynk && ./install.sh
 
✓ Installed to ~/.synlynk/bin/synlynk
✓ PATH updated in ~/.zshrc
✓ synlynk 0.4.1 ready
2
Bootstrap any project
Run inside any repo. Creates project-docs/, writes agent instruction files for all detected IDEs and CLIs, and generates a SHA manifest. Safe to re-run — appends to existing files without overwriting user content.
synlynk init
~/my-project $ synlynk init
 
Mode: single | team? single
 
✓ project-docs/roadmap.md
✓ project-docs/memory.md
✓ project-docs/todo.md  & costs.md
✓ CLAUDE.md · GEMINI.md · AGENTS.md  (trio instruction files)
✓ .cursor/rules/synlynk.mdc · .windsurfrules  (IDE reach)
✓ Agents: claude · agy · codex
 
→ synlynk exec claude
3
Run your first context-injected session
Synlynk prepends your project context — the AI already knows your roadmap, decisions, and open tasks before you type a word.
synlynk exec
$ synlynk exec claude
 
📋 Context injected   (memory · roadmap · todos)
💰 Budget: $2.34 / $10.00 used this session
🚀 Launching claude...
 
claude › I can see you're building a billing API for the
monetization module. Based on your roadmap...

Chapter 03 — Multi-Agent Dispatch 04
v0.4.0 · Hybrid Workgroup Bootstrap

Run agents in parallel.
In the background.

Dispatch fires an agent as a background job — captured to a log file, tracked by the job store. Multiple agents run simultaneously while your terminal stays free.

synlynk dispatch
$ synlynk dispatch claude --task "Write unit tests for the billing module"
✓ Dispatched job-a3f2  (claude)  running in background
 
$ synlynk dispatch agy --task "Review billing API for edge cases"
✓ Dispatched job-b7e1  (agy)  running in background
 
# Both agents working simultaneously — terminal is yours

Monitor all jobs

synlynk jobs
$ synlynk jobs
 
● job-a3f2  claude  running 2m14s
● job-b7e1  agy  running 1m58s

Tail a job's output

synlynk logs
$ synlynk logs job-a3f2

✓ test_charge_success
✓ test_refund_partial
→ 12 / 12 tests passing

Chapter 04 — Safety Systems 05
Sentinel & Budget Guard

Your safety net,
always on.

Synlynk watches every agent invocation for danger signals and surfaces alerts before they become expensive mistakes.

Budget Guard

Set spending limits in .synlynk/config.json. Synlynk blocks the next exec the moment you'd exceed your limit. Cost shown after every run.

.synlynk/config.json
{
  "limit_usd": 10.00,
  "limit_requests": 100
}
budget pulse
── Budget Pulse ──────────────
Cost:     $2.34 / $10.00
Requests: 14 / 100
Tokens:   2,840 in · 480 out

Sentinel Patterns

Sentinel watches your telemetry log and fires alerts automatically for four danger patterns.

FLATLINE
Same command failed 3× in a row
Possible hallucination loop. Manual intervention recommended before more tokens burn.
LOOP
Same command succeeded 5× in <10 min
Likely an automated loop running unproductively. Investigate before continuing.
QUOTA
Quota exhaustion detected in output
Output matched a known quota-exhausted phrase. Switch agent or check plan limits.
ZOMBIE
Job PID dead but never marked complete
Or job has been running >30 min without output (STALL). Reconciled automatically on next run.
DRIFT
Instruction file edited outside synlynk
A tracked file's synlynk section was modified externally. Run synlynk instructions update or ack to resolve.

Managing Alerts

sentinel commands
$ synlynk sentinel list
 
[CRITICAL]  FLATLINE      `synlynk exec claude` failed 3x  — 2026-06-14 09:41
[WARN]   SUCCESS_LOOP  Same command 5x in 4.2 min     — 2026-06-14 10:02
 
$ synlynk sentinel clear --code FLATLINE
✓ Cleared FLATLINE alerts

Chapter 05 — Command Reference 06
All Commands

Everything
at a glance.

Setup & Context

CommandWhat it does
synlynk initBootstrap project-docs/, agent instruction files, and .synlynk/config.json in any repo. Safe to re-run — skips existing files.
synlynk exec <agent>Run an agent interactively with full context injection (memory + roadmap + todos prepended). Captures token counts and cost on exit.
synlynk checkpointForce a context snapshot — regenerates .synlynk/context.md from current project-docs state.
synlynk statusDashboard showing current project state, recent costs, active sentinel alerts, and running jobs.
synlynk upgradeCheck for and apply updates from the remote repository.

Dispatch & Jobs v0.4.0

CommandWhat it does
synlynk dispatch <agent>Fire an agent task in the background. Captures stdout to a log file. Returns a job ID immediately. Accepts --task "..." and --story <id>.
synlynk jobsList all tracked jobs with status (running / completed / failed), agent, and duration. Use --all to include completed jobs.
synlynk logs <job-id>Tail the output log of a specific job. Use --tail N to control how many lines to show.
synlynk run --trioRun a three-phase Architect → Builder → Verifier pipeline with context hand-off between each phase.

Sentinel

CommandWhat it does
synlynk sentinel listShow all active alerts with severity, code, message, and timestamp.
synlynk sentinel clearClear alerts. Target with --severity CRITICAL or --code FLATLINE.

Instructions v0.4.1

CommandWhat it does
synlynk instructions statusShow status of each tracked instruction file: ok / missing / drifted / untracked / no-markers.
synlynk instructions diffShow user content outside synlynk markers in each tracked file.
synlynk instructions update <file>Regenerate the synlynk section in a file and reset its SHA in the manifest.
synlynk instructions ack <file>Remove INSTRUCTION_DRIFT lines from sentinel.md without regenerating.

.synlynk/
  ├── config.json     ← budget limits
  ├── context.md     ← auto-generated
  ├── telemetry.json  ← rolling log
  ├── instructions.json ← SHA manifest
  └── logs/          ← job outputs

What agents receive

Every synlynk exec prepends memory.md + roadmap.md + todo.md as a single context snapshot before handing off to the agent.

Keep project-docs current
Synlynk's context is only as good as your memory.md and roadmap.md. Update them after significant decisions — agents pick up changes on the next exec automatically.
Smart routing is live in v0.5.0
synlynk dispatch and synlynk run --trio now route automatically based on capability scores. Use synlynk story create and synlynk score add to build up routing data across your team.
