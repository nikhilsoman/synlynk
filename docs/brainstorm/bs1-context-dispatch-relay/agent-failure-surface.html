OB-7 · CA-2 · DevX-3 · AB-4 — one unified design for the dispatch lifecycle

✈️
Phase 1: Pre-flight checks
Before any process is spawned — OB-7 (agent availability) + CA-2 (pre-req audit)
OB-7 CA-2
Happy path
terminal
synlynk dispatch --agent agy --task "review PR 42"
↳ preflight: daemon alive · agy available · budget ok
✓ dispatched — job 7f3a (agy)
Agent unavailable
terminal
synlynk dispatch --agent agy --task "review PR 42"
⚠ preflight: agy not running on this machine
Last seen: never installed
Install: claude extensions install agy
Or use: --agent claude
Budget blocked
terminal
synlynk dispatch --agent claude --task "refactor auth"
✗ preflight: budget limit reached
Used: $4.87 / $5.00 limit
Raise: synlynk config set limit_usd 10
Override: --force-dispatch
Pre-flight gate: check daemon alive → agent installed → agent not throttled → budget headroom. Each check is fast and silent on pass. Only prints when something is wrong. The "↳ preflight:" line shows in --verbose mode only.

DevX-3 — what observability does dispatch carry? What prints, when?

Background mode (default)
terminal
synlynk dispatch --agent claude --task "review PR 42"

ID AGENT STORY STATUS AGE COST
7f3a claude #42 running 1m23s $0.004

Budget Pulse prints after every job (current behavior, kept). The new synlynk jobs command is the "jobs board" — replaces asking "what's running?" to an agent. Context size shown in the header line: context: task (820 bytes) — this closes DevX-1/Q9 (context size budget visibility).

Phase 3: Failure and sentinel alerts
AB-4 sentinel extension + CA-2 compliance tags visible in failure output
AB-4 CA-2 DevX-3
Job failed — exit non-zero
terminal
synlynk jobs --watch
✗ 7f3a · claude · failed (exit 1, 4m22s)
Error: agent exited non-zero
Logs: synlynk logs --job 7f3a
Retry: synlynk dispatch --retry 7f3a
Sentinel: FLATLINE
terminal
⚑ SENTINEL · FLATLINE
3 consecutive failures: jobs 7f3a 7f3b 7f3c
Agent: claude · Task pattern: "review PR *"
Action: dispatch paused (5m cooldown)
Dismiss: synlynk sentinel ack flatline
Written: .synlynk/sentinel.md
Sentinel: VERIFY_SKIP (AB-4)
terminal
⚑ SENTINEL · VERIFY_SKIP
Job 7f3c: instruction compliance: LOW
Missing tags: verify_before_commit
             check_test_run
Details: synlynk jobs --compliance 7f3c
Key design: failure output always includes the retry or dismiss command inline — no hunting. FLATLINE auto-pauses dispatch (cooldown); VERIFY_SKIP is informational only (for now). The compliance tags bridge AB-4 (instruction adherence) and CA-2 (audit) — same compliance_tags field on the job_completed relay event.

Context size inline visibility (Q9)
No separate command — surfaces in dispatch output and jobs board
Context size warn (> 80% of budget)
terminal
synlynk dispatch --agent claude --task "refactor auth"
⚠ context: full (94KB) — exceeds soft limit (80KB)
Use --context-mode task to reduce to ~820 bytes
Use --context-mode none to suppress entirely
✓ dispatching anyway — job 8a1c · override: add context_tokens_max to config
Context size blocked (--strict flag or config hard limit)
terminal
synlynk dispatch --agent agy --task "review all PRs"
✗ context: full (119KB) — hard limit is 100KB
Set in config: context_tokens_max: 100000
Options:
--context-mode task → ~820 bytes
--context-mode none → 0 bytes
--override-context-limit (unsafe)
Soft vs hard limit: soft limit warns but continues; hard limit blocks. Default: warn at 80KB, no hard block (too strict for current usage). Config: context_tokens_max for hard block, context_warn_bytes for soft warn. This closes Q9 without adding a separate preflight command.
Cluster A design principle: the agent lifecycle has four moments that matter — pre-flight, job start, job done, job failed. Each moment gets one line of output by default. Verbose flag adds detail. Sentinel fires an additional alert line asynchronously. All four points write structured data to the relay (job_dispatched / job_completed / alert_raised) — the terminal output and the relay event are generated from the same object. No dual bookkeeping.
