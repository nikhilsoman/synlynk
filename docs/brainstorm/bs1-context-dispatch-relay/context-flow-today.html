Step 2 — generate_context()
Scope selection Problem #1
scope = "full" because story_id is None
If story_id were set: scope = "task:<story_id>" → slim context
Without it: full project-docs blob every time
Root cause: cross-repo dispatch has no story_id. No mechanism to say "suppress project-docs — task is self-contained." Falls through to full context unconditionally.

Context size
119 KB — 300× what the task needs
~400 bytes — task + agent framing + capability

What already works

  • Per-job prompt files at .synlynk/prompts/
  • Per-job log files at .synlynk/logs/
  • _generate_task_context() exists for story-scoped dispatch
  • Agy truncation partially limits token waste
  • 64 KB warning fires (not a silent failure)

Three problems to fix

  • #2 Race: global context.md shared across concurrent jobs
