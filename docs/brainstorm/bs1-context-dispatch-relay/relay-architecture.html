Event schema — typed core + generic tail
story_updated
story_id, status, agent,
title, engg_domain, ts, origin_node
job_dispatched
job_id, agent, story_id,
priority, depends_on[], ts, origin_node
job_completed
job_id, exit_code, duration_s,
tokens_in, tokens_out, ts, origin_node
alert_raised
pattern, severity, job_id?,
message, ts, origin_node
context_checkpoint
context_hash, size_bytes,
ts, origin_node
table_changed  // generic fallback
table, op (INSERT|UPDATE|DELETE),
pk, changes{}, ts, origin_node
Why typed + generic: 5 typed events cover 95% of relay traffic with semantic meaning receivers can act on. table_changed covers schema additions without relay code changes — adding a new table doesn't require a new event type until it's worth promoting.
Deployment mode for v0.9.4

LAN — mDNS

Daemons announce via Bonjour/Avahi. Peer-to-peer, no internet dependency. One daemon is "host."

Zero config, zero cost
Works completely offline
Same network only
Host machine must stay up

Requires cloudflared installed

Dedicated relay server. All nodes connect to a fixed address. Required for enterprise / air-gapped.

Full control, no third-party
Persistent — relay survives node restarts
Cost + maintenance
