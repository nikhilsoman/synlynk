synlynk state: local-first, sync is optional

Every tier works standalone. Upper tiers add reach, not dependency.

Tier 0 — Solo / Offline / Air-gapped ✓ always works
~/.synlynk/projects/<hash>/state.db ~/.synlynk/projects/<hash>/context.md ~/.synlynk/projects/<hash>/memory.md ~/.synlynk/projects/<hash>/devlog.md
SQLite is the source of truth. Markdown files are generated views — written by synlynk, readable by humans, editable offline (changes sync back to DB on next run).
No git involvement. No conflicts. No network.
Human asks their preferred agent anything → agent reads from state.db + context.md. Other agents are invisible workers.
↕ optional sync
Tier 1 — Team (relay, v0.9.3) optional graceful degradation ↓ Tier 0 on failure
relay.synlynk.com (community) self-hosted --enterprise LAN mode (no internet)
Relay syncs state.db deltas between teammates over WSS/443. Each developer's machine is authoritative for their own writes; relay is a broadcast bus, not a source of truth.
If relay goes down: each machine continues working fully on Tier 0. State.db accumulates local changes. On reconnect, relay merges deltas (last-write-wins per story_id, conflict-free for append-only tables like capability_ratings + devlogs).
If security blocks relay: LAN mode uses mDNS discovery + direct WSS on local network. No internet required. Air-gapped teams use git transport (see below).
↕ optional sync
Tier 1b — Air-gapped / relay-blocked teams: git transport fallback
For teams whose security policy blocks all relay options: synlynk exports a compact state delta bundle (JSON, not markdown) that can be committed to a separate synlynk-state branch or sidecar repo. Teammates pull and import. Same conflict-free merge model as relay, just async via git instead of real-time WSS.
No markdown files in git. No conflicts. The bundle is binary-safe and opaque to git diff.
↕ optional
Tier 2 — Workspace (multi-repo, v0.10.0) v0.10.0
Relay scope expands: stories and capability ratings federate across repos in a workspace. Cross-repo epics live in a workspace state.db. Each repo's local Tier 0 stays independent; workspace relay adds a coordination layer above it.
↕ optional
Tier 3 — Community / Tokq (v1.1–v1.3) v1.1+
Capability ratings (Ed25519-signed at Tier 0) propagate to the community server. Cross-workgroup agent reputation emerges from aggregated, tamper-evident ratings. Tokq bridge at v1.3 makes these ratings portable across organisations. The signed ledger that starts at Tier 0 becomes the community trust anchor.
Key principle: local-first, sync is additive
Every tier works without the tier above it. Relay failure → Tier 0, not broken.
Security blocks relay → Tier 1b git transport or stay on Tier 0.
Community server down → workspace, team, solo all unaffected.
The human never sees any of this. They just talk to their agent.
