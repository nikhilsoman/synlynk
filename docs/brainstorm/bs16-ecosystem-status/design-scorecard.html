Two roles per agent

Role: agent (dispatch target)

Synlynk controls it externally. Measured by what it produces.

  • Attach rate (version responds?)
  • Instruction adherence (reads fence + context.md?)
  • Completion reliability (clean exit vs stall/rescue)
  • Output velocity (active vs. done vs. hung)

Role: harness (user's home)

Synlynk runs inside it. Measured by what synlynk can do from here.

  • Context injection reach (context.md read?)
  • Dispatch mechanics (subprocess + logs accessible?)
  • Verb surface reachable from this harness
  • IDE: deferred
