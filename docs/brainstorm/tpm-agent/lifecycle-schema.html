Lifecycle as First-Class Entity

A typed, configurable artifact chain. Each stage is an agent attachment point that produces an actionable artifact.

Lifecycle Type Schema
{
  "id": "software-development",
  "label": "Software Dev",
  "stages": [
    {
      "id": "brainstorm",
      "artifact_out": "session_transcript",
      "agent_roles": ["architect"],
      "consent": "approve",
      "advance": "manual"
    },
    {
      "id": "spec",
      "artifact_in":  "session_transcript",
      "artifact_out": "spec_doc",
      "agent_roles": ["architect", "reviewer"],
      "advance": "on_file: docs/superpowers/specs/*.md"
    }
  ]
}
Stage fields
artifact_in / out — what the stage consumes / produces
agent_roles — which roles can act here
consent — auto / notify / approve
advance — what triggers next stage
advance triggers
manual — human calls lifecycle advance
on_file: glob — artifact file appears
on_exit_0 — shell command exits clean
on_db: table.status — DB row state changes
