WEEK 4
v0.9.3
Workgroup Relay — Revolving Host Model
• Stateless relay — events stored in each member's state.db, relay just routes
• Any active member can be host — relay bootstraps from any online member's state
• Revolving host protocol: loud broadcast on host change — all daemons notified, grace period (old host stays alive 10 min), new URL announced via signed message
• Offline resilience: local-first queue, events flush on reconnect
• Self-hosted v0.9.3 → VPS migration: change one URL, zero data migration (relay is stateless)
• synlynk relay start/status/handover <member>

📡 Relay VPS Options (separate deep-dive brainstorm)
Fly.io
Best for small teams — free tier covers relay, edge deployment, ~$5-10/mo paid. Simplest path.
Hetzner Cloud
Cheapest raw compute (~$3-5/mo 1-core VM). Self-managed but total cost winner for persistent relay.
E2B / Modal
AI-native agent infra. E2B better for persistent relay; Modal better for event-triggered agents. Higher cost.
Railway + Render also viable. Deep dive: architecture, pricing, agentic ops features, migration path from self-hosted → VPS.
🗳 Consensus Decision Framework — first-class in v0.9.1
synlynk decide "<topic>" --panel claude,agy,codex [--record]
→ Convenes panel · records each participant's input · synthesizes into a signed Decision record
→ Decision record: immutable, append-only, signed by all participants, linked to resulting artifacts (spec, roadmap, todo)
→ Lives in project-docs/decisions/YYYY-MM-DD-<slug>.md + indexed in state.db
→ A Decision is a peer to Epic in the PM hierarchy — deliberation that produces direction, not work items
→ This roadmap realignment is the first Decision record to be created
