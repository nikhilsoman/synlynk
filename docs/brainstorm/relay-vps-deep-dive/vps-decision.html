synlynk Relay — Ownership Decision

Decided 2026-06-21 · workgroup (Nikhil + Claude)

Decision
Hybrid — Community-First (C, B-leaning)
Community relay (B) is the default experience. Self-provisioned VPS (A) is gated behind enterprise/exception. The progression: LAN → Cloudflare Tunnel → Community Relay (default) → Self-Hosted (enterprise).

Three Relay Tiers

Default · All users
Community Relay
Hosted by synlynk · relay.synlynk.com
  • Free for workgroups (up to N members)
  • Namespace isolation per workspace
  • WSS/443 · Let's Encrypt · global Fly.io edge
  • synlynk relay join — one command
  • No VPS to manage
  • Drives network effect + moat data
Enterprise / Exception
Self-Provisioned VPS
Fly.io · Hetzner · bring your own
  • Full data sovereignty
  • Custom domain + private namespace
  • synlynk ships ready-made fly.toml
  • Same protocol — all clients compatible
  • Target: regulated industries, air-gapped
🔒 Feature gate: requires --enterprise flag or explicit exception token
v0.9.3 prototype
LAN / CF Tunnel
Self-hosted · no VPS · local only
  • Day-1 option during v0.9.3 build
  • Good for fully local teams on VPN
  • Graduates to Community Relay via synlynk relay upgrade
  • Always available as fallback

UX Flow — What users see

1
synlynk relay join
Default path. Auto-connects to relay.synlynk.com. Namespace = workspace ID. No config needed.
2
Team members join
synlynk relay join --workspace <name> — member connects to same namespace. Relay authenticates via Ed25519.
E
Enterprise / Exception path (gated)
synlynk relay self-host --fly or --hetzner — deploy your own relay. Requires enterprise flag or exception token from synlynk.com.
# Default — community relay (what 95% of users see
$ synlynk relay join

# Output:
✅ Connected to relay.synlynk.com/ws/workspace-abc123
3 members online · host: nikhil@machine · WSS/443
# Enterprise / exception path (gated)
$ synlynk relay self-host --fly --enterprise
# deploys fly.toml to user's Fly.io account
# requires SYNLYNK_ENTERPRISE_KEY or exception token

Version Map

v0.9.3 Relay prototype LAN + CF Tunnel mode. Community relay endpoint stubbed (connects to future relay.synlynk.com). --self-host works ungated.
v1.0.0 Community relay live relay.synlynk.com on Fly.io. Default for all new workgroups. Self-host gated behind enterprise flag.
v1.1.0 Community server Relay graduates to persistent community server. Workgroup namespaces → team namespaces. Network effect on routing graph accelerates.
⚠️ Feature gate design (v0.9.3 → v1.0.0 transition): the synlynk relay join command already points at relay.synlynk.com even in v0.9.3, but shows a "community relay launches with v1.0 — using LAN mode for now" message until the server is live. No UX change at v1.0 launch — just the server turning on.
