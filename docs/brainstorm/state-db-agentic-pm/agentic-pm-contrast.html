Human PM vs. Agentic PM — The Core Shift

What anchors project structure when there is no human cadence?

Human PM anchors on time + capacity
Sprint 3 · Week of June 9
← time is the primary container

Story points: 8
← effort estimate as unit of work

"Done" = sprint ended + accepted
← temporal ritual defines completion

Velocity / burndown
← progress measured in time consumed

Blocking = waiting on a human
← idle time is the bottleneck
Agentic PM anchors on dependencies + verification
Phase: SQLite kernel
← capability gate is the container

Parallelism surface: 4 stories
← concurrent work is the unit of throughput

"Done" = verification criteria pass
← deterministic signal defines completion

Dependency graph / critical path
← progress measured in gates cleared

Blocking = unmet dependency
← logic is the bottleneck, not time

Proposed agentic hierarchy

Project → Phase → Epic → Story → Event
Project   synlynk
↳ the stable root. never changes.

Phase    "SQLite Kernel"  ← was: v0.5.0
↳ a capability gate. after this, X becomes possible.
↳ has: capability_assertion, depends_on_phase
↳ human alias: "v0.5.0", "Aug 2026" — optional labels only

Epic     "Context Bridge"  ← agent domain cluster
↳ a coherent cluster routable to one agent or domain.
↳ defines the parallelism surface within a phase.
↳ optional for small projects; essential for team builds.

Story    "migrate project-docs to state.db"  ← was: task
↳ atomic deliverable. one verifiable done state.
↳ has: done_criteria, depends_on (other story IDs)
↳ no time estimate. no story points. just: done or not.

Event    "state.db created · schema v1 applied"  ← was: devlog
↳ append-only state change. never edited, never deleted.
↳ types: started · completed · blocked · decision · note
↳ devlog becomes a filtered view of events. not a separate store.

What collapses

roadmap + tasks → phases + epics + stories. The roadmap table was a version-level alias for what is actually a dependency graph of capability gates. Tasks were time-anchored work items. Stories replace them with dependency-anchored, verifiable deliverables.

devlog → events. Session narrative (human temporal) becomes state transition log (agentic logical). A devlog view is just SELECT * FROM events WHERE user='nikhil' AND date >= 'today-7d'.
