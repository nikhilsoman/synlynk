New CLI surface

Read commands

# human-readable markdown to stdout

synlynk context show tasks
# filtered by table

synlynk context --for claude
# agent-filtered view (v0.8 alignment)

Write commands

synlynk log "message" --agent claude
# append devlog entry

synlynk task done "task text"
# mark task complete

synlynk memory set key value
# upsert memory entry

What "nightly consolidation" becomes

The DB is always live — no batch consolidation needed

The original "nightly consolidation" idea assumed markdown files as primary store and SQLite as a derived index. With state.db as primary, every synlynk exec writes directly — the DB is always current.

What remains useful on a schedule:

  • Vacuum — SQLite WAL compaction, once nightly
  • Devlog archive — entries older than 30 days → devlog_archive table
  • Costs rollup — weekly aggregate rows to keep costs table lean

These are maintenance ops on state.db, not a batch import from files.

Agent experience — unchanged on the surface

Agents still see .synlynk/context.md injected before each run. The change is invisible to them — but the state is now always consistent, never a stale worktree snapshot.
