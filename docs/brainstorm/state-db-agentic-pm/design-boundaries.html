Table Boundaries — Verified Against generate_context()

What each table stores, how it feeds context, and corrections to the draft schema

memory → memory table

⚠ schema correction

Scope: Durable architectural and positioning decisions. Injected in full into every context snapshot.

Structure in practice: Thematic sections (## Positioning, ## Architecture) with multiple narrative bullets per section. Not key/value pairs.

Correction: Replace key TEXT UNIQUE / value TEXT with heading TEXT UNIQUE / body TEXT — store at section granularity, not bullet granularity. Preserves the narrative structure agents expect.

id INTEGER PK | heading TEXT UNIQUE | body TEXT | agent TEXT | updated_at TIMESTAMP

todo.md → tasks table

⚠ schema correction

Scope: Atomic work items. Context injection: unchecked - [ ] lines only — completed tasks are filtered out.

Structure in practice: Grouped by milestone version (## v0.4.0), with embedded <!-- id: N --> and [@username] attribution.

Correction: Replace priority INTEGER (doesn't exist) with milestone TEXT (e.g. "v0.4.0"). Promote to task_id INTEGER primary key.

task_id INTEGER PK | text TEXT | status TEXT | owner TEXT | milestone TEXT | created_at TIMESTAMP

roadmap.md → roadmap table

⚠ schema correction

Scope: Version-level milestones. Context injection: only rows with status "In Progress" or "Next".

Structure in practice: 6-column table — version | theme | os_layer | infrastructure | status | target.

Correction: Add os_layer TEXT and infrastructure TEXT — both are currently in the table and matter for context generation. Replace notes TEXT (not in current doc).

id INTEGER PK | version TEXT UNIQUE | theme TEXT | os_layer TEXT | infrastructure TEXT | status TEXT | target TEXT

devlogs/* → devlog table

✓ clear, minor addition

Scope: Session narrative log. Context injection: own last 7 days, teammates' last entry.

Structure in practice: Date-grouped sections per user file (nikhil.md, etc). Session title + narrative content.

Addition: user TEXT (git username, maps to devlogs/<username>.md) is already implied but should be explicit. Rename agent → user for clarity — in synlynk, the devlog author is always a human user, not a model.

id INTEGER PK AUTOINCREMENT | date TEXT | session_title TEXT | user TEXT | content TEXT | ts TIMESTAMP

costs.md → costs table

✓ clear, no change

Scope: Session cost ledger. Not injected into context — purely for human review and budget checks.

Structure in practice: Date | User | Requests | Tokens In/Out | Estimated Cost | Summary.

No change needed. Schema is correct. Budget checks (check_budgets()) query costs by summing cost_usd.

out of scope for state.db

Ephemeral — written and cleared by check_flatline() and check_budgets(). No historical value. Stays in .synlynk/sentinel.md as today.
