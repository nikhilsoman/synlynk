Agent Architecture — Archetypes × Levels

Same archetype, deployed at any level — scope defines what it monitors, executes, and communicates

Three core archetypes (v0.8.x series — ship rapidly)
🔧 Maintainers
Self-healing · SOP keepers · Environmental health
✅ Support Engineer — tests, sentinel, telemetry, costs, GitHub issues
v0.8.1 Security Guard — dep vulns, secret leaks, CVEs
v0.8.2 Compliance Officer — convention drift, instruction staleness
later Incident Responder — live issue triage + RCA
📣 Communicators
Content · Marketing · Community voice
v0.8.3 Marketing Intern — blog, release notes, social copy
v0.8.4 Docs Keeper — keeps docs current with code changes
later Community Manager — GitHub discussions, issue responses
later Changelog Publisher — auto release notes
🎯 Orchestrators
Work management · Signals · Cross-agent coordination
v0.8.5 PM Agent — growth signals, story routing, sprint pulse
v0.8.6 Growth Scout — stars, clones, adoption signals
later Sprint Captain — cross-repo epic coordination
later Budget Sentinel — token quota across team/workspace
Deployment levels — same archetype, scoped to context
Level 🔧 Maintainers monitor… 📣 Communicators send to… 🎯 Orchestrators manage…
🏠 Workgroup
human + agents + autopilots, 1 project 		Tests · sentinel alerts · telemetry · per-repo costs · open issues GitHub repo · PR descriptions · project blog · devlogs Stories · epics · routing · token budget per story
👥 Team
across workgroups, same product/multi-repo 		Cross-repo health · integration failures · drift across repos · shared capability drops Team channels (Slack/email) · cross-repo status digest · public changelog Cross-repo epics · team token budgets · agent routing across workgroups
🏢 Enterprise Workspace
across teams, org-wide 		Org-wide cost burn · compliance across repos · security posture · capability benchmarks Leadership dashboards · org-level blog · compliance reports Org portfolio · cross-team budget allocation · agent entitlements
🌐 Domain/Community
discipline or functional domain 		Domain health signals · community adoption · best-practice drift Public community channels · domain newsletters · Tokq marketplace Convention packages · shared routing intelligence · community epics
Context portability — the key design principle
A Support Engineer in the synlynk repo monitors synlynk's health.
The same Support Engineer archetype deployed in a customer's Superagents workgroup monitors their project's health.
At team level it monitors cross-repo integration health. At workspace level, org-wide posture.
The archetype is the pattern. The .agents/<name>.json config + deployment scope give it its specific purpose.
