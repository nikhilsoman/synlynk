Design — Section 1: Identity Model

Two layers compose to answer: who can do this work, and who does it best?

Identity stack — from trust primitive to routing decision
Local Identity   Ed25519 keypair · agent_uuid · git_user
↳ cryptographic anchor. generated once at synlynk init. machine-scoped.
↳ already designed (Tokq Gap 1) — pulling forward from v0.9.0 to now.
↳ signs dispatch events. proves "this synlynk instance did this work."

Role           Architect · Builder · Verifier (+ custom roles)
↳ primary entitlement unit. defines what work can be claimed and what it can touch.
↳ a story's role requirement is set at epic creation (writing-plans output).
↳ entitlements attach here: what paths, what actions, what approval threshold.

Agent Profile   cli × model × environment × competency_scores
↳ fitness function. who fills a role best right now?
↳ cli: claude-code | claude-desktop | gemini-cli | codex
↳ environment: what native actions this cli can perform (see Section 2)
↳ competency_scores: earned from past executions per (agent, domain) — v0.5.0

Dispatch Decision   role match + entitlement check + profile fitness + quota
↳ role defines the gate. profile picks the winner within the gate.
↳ "Builder needed, domain=api → claude-code (score:0.91, quota:80K, cost:$0.18)"

How the layers answer different questions

Role answers

  • Can this story auto-dispatch?
  • What files/paths can be written?
  • Can it create branches? Merge PRs?
  • Does it need human approval first?
  • What quota ceiling applies?

Agent Profile answers

  • Which CLI fills this role best today?
  • Is bash execution available?
  • Can it open a browser / use GUI?
  • What's the cost per token for this model?
  • Has it done this domain before?

Key invariant — roles are stable, profiles are dynamic

Roles are defined once per project and change rarely. The Builder role's entitlements (can write src/, can create branches, cannot merge to main) are stable architectural decisions.

Agent Profiles change constantly — competency scores update after every execution, quota headroom changes hourly, new CLI versions add environment capabilities.

Consequence: entitlement enforcement uses the stable role definition. Routing uses the live agent profile. These two concerns never mix.
