Wizard step sequence — which order?

Same steps, two orderings. Auto = detected silently. Prompt = user confirms/overrides. Scan = spawns synlynk scan inline.

auto detected silently, shown for confirmation
prompt user must confirm or type
scan spawns synlynk scan subprocess

A — Detect first, configure second

  • 1
    Home harness detectionauto
    Inspect $TERM_PROGRAM, $CLAUDE_TRACE, process parent. Show detected harness, ask to confirm.
  • 2
    Agent discoveryauto
    which claude / codex / agy / grok — show found/not-found list, ask to confirm.
  • 3
    Workspace nameprompt
    Infer from git remote or dir name, user confirms or types.
  • 4
    Agent rolesprompt
    Show default role table (Claude=PM, Agy=impl, Codex=impl, Grok=impl). Accept or customise.
  • 5
    Write config + directive filesauto
    .synlynk/config.json, CLAUDE.md ##Your Role, GEMINI.md, AGENTS.md written.
  • 6
    Run synlynk scan?prompt
    Offer to scan repo now. Y runs it inline. N prints reminder to run manually.
  • 7
    First dispatch cheat sheetauto
    Print 5-line quickstart: synlynk dispatch, synlynk probe, synlynk status.
