DevX gaps — not covered by any story yet

Repo scan + context build

synlynk init copies templates but never looks at the repo. It doesn't know the stack, existing README, open PRs, or whether there's already a CLAUDE.md. Context.md is hand-crafted from whatever the user put in project-docs/.

Agent discovery

There is no auto-detection. User manually creates .agents/claude.json, .agents/agy.json etc. A new user has no idea what's installed or how to get started dispatching.

Home harness discovery

synlynk doesn't know which CLI it's running inside. Claude Code? Codex? Terminal zsh? The home harness is the source of `synlynk exec` — but it's never detected or recorded.

FTUE TUI wizard

synlynk init dumps files silently. A first-time user has no guided path from "installed" to "first dispatch." No confirmation, no health check, no "here's what synlynk found."

Today's brainstorm goal

Design the full FTUE + onboarding system: what does synlynk init actually do, what does the TUI wizard look like, how does agent + harness discovery work, what is the repo scan scope, and how do these land as stories in v0.10.0.
