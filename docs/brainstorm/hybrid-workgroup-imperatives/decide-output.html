synlynk decide — Decision record format

When the panel reaches consensus, a Decision record is written to project-docs/decisions/. Which format?

A

Markdown with YAML frontmatter

Human-readable, git-diffable, consistent with existing project-docs/ style. Ed25519 signatures stored as a signatures: block in frontmatter.

project-docs/decisions/2026-06-22-relay-ownership.md
---
decision_id: dec-a1b2c3d4
topic: "Relay ownership model"
date: 2026-06-22
panel: [claude, agy, codex, nikhil]
status: approved
signatures:
  claude:  "ed25519:abc123..."
  agy:     "ed25519:def456..."
  codex:   "ed25519:ghi789..."
---

## Topic
Relay ownership model for v0.9.3

## Panel Inputs

### claude
Community relay as default; self-host gated behind
enterprise flag. Fly.io for infrastructure...

### agy
Agree with community-first. Cloudflare Tunnel as
the bridge mode for hybrid teams...

### codex
LAN mode must remain available offline...

## Synthesis
Community relay is the default experience...

## Decision
Option C (Hybrid): community relay default,
self-host gated.
B

JSON record

Machine-readable, easy to query from state.db or scripts. Signatures are first-class fields. Less readable in GitHub PR reviews.

project-docs/decisions/2026-06-22-relay-ownership.json
{
  "decision_id": "dec-a1b2c3d4",
  "topic": "Relay ownership model",
  "date": "2026-06-22",
  "panel": ["claude", "agy", "codex", "nikhil"],
  "status": "approved",
  "inputs": {
    "claude": "Community relay as default...",
    "agy": "Agree with community-first...",
    "codex": "LAN mode must stay available..."
  },
  "synthesis": "Community relay is the default...",
  "decision": "Option C (Hybrid)",
  "signatures": {
    "claude": "ed25519:abc123...",
    "agy": "ed25519:def456...",
    "codex": "ed25519:ghi789..."
  }
}
C

Markdown + sidecar JSON (Recommended)

Human-readable .md for reading and git review, .json sidecar for machine querying. Same decision_id ties them together.

Two files, one decision
project-docs/decisions/
  2026-06-22-relay-ownership.md   ← human readable
  2026-06-22-relay-ownership.json ← machine queryable

Both share decision_id: dec-a1b2c3d4
Signatures live in the JSON.
