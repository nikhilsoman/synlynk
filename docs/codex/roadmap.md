# Native Codex Support Roadmap

## Objective

Make Synlynk a reliable local session ledger and context compiler for Codex users across CLI, Desktop App, and third-party IDE shells.

## v0.3.0: Codex-Native Setup

Goal: Codex users should get the right instructions automatically after `synlynk init`.

Deliverables:

- Generate `AGENTS.md`.
- Add `synlynk init --codex`.
- Document Codex CLI interactive usage.
- Document Codex Desktop baseline behavior.
- Update README to distinguish Codex-native instructions from generic `AI_INSTRUCTIONS.md`.

Success criteria:

- New projects contain `AGENTS.md`.
- Existing projects can refresh Codex files with `synlynk init --force` or `synlynk init --codex`.
- Codex users no longer need to copy/paste session instructions manually.

## v0.4.0: Diagnostics and Reliability

Goal: users can verify whether Codex support is configured correctly.

Deliverables:

- Add `synlynk doctor --codex`.
- Add config validation.
- Detect missing/stale context.
- Detect malformed `costs.md`.
- Detect missing or weak `AGENTS.md`.
- Add fake-Codex subprocess tests.

Success criteria:

- `synlynk doctor --codex --json` is stable enough for bug reports.
- Codex setup failures are actionable.
- CI does not require a real Codex install.

## v0.5.0: Codex Non-Interactive Runner

Goal: Synlynk can run repeatable Codex automation through structured output.

Deliverables:

- Add `synlynk codex "<prompt>"`.
- Run `codex exec --json --cd <repo>`.
- Preserve Codex child exit codes.
- Add passthrough for model, profile, sandbox, and advanced flags.
- Add stdin prompt support.
- Emit structured `codex_exec` telemetry.

Success criteria:

- A fake Codex JSONL stream is parsed in tests.
- Non-zero Codex exits propagate.
- Users can run repeatable summarize/review tasks without manual wrapper setup.

## v0.6.0: JSONL Usage and Cost Ingestion

Goal: capture Codex usage data when structured events make it available.

Deliverables:

- Add tolerant Codex JSONL parser.
- Extract final messages, usage data, tool events, and errors.
- Append cost rows only when data is available.
- Store token-only records when dollars are unavailable.
- Update `synlynk status` to reflect Codex usage.

Success criteria:

- Parser survives malformed and unknown events.
- Synlynk never invents cost values.
- Cost and telemetry outputs remain parseable.

## v0.7.0: Codex Desktop App Helpers

Goal: improve Codex Desktop App ergonomics without depending on unstable app internals.

Deliverables:

- Add Desktop setup docs.
- Add optional local action recommendations.
- Add guarded deeplink helper if supported and verified.
- Report `.codex/` project config presence in `doctor --codex` without requiring it.

Success criteria:

- Desktop users have clear setup instructions.
- No unverified `.codex/` files are generated by default.
- Desktop helpers are opt-in and reversible.

## v0.8.0: Third-Party IDE Hardening

Goal: make Codex support reliable from editor terminals and task runners.

Deliverables:

- Document VS Code, Cursor, Windsurf, and generic terminal usage patterns.
- Add `synlynk doctor --codex --ide` checks where feasible.
- Add environment diagnostics for PATH, cwd, git root, and writable state.
- Add examples for invoking `synlynk codex` from tasks.

Success criteria:

- IDE users can diagnose path/config mismatches.
- Synlynk behavior remains portable across editor shells.

## 1.0.0: Stable Codex Contract

Goal: freeze the local Codex integration contract.

Deliverables:

- Stable `AGENTS.md` schema.
- Stable `doctor --codex --json` schema.
- Stable `codex_exec` telemetry event shape.
- Migration path for old Synlynk projects.
- Full documentation and support playbook.

Success criteria:

- Users can safely automate Codex workflows on top of Synlynk.
- Breaking changes require explicit migration notes.
