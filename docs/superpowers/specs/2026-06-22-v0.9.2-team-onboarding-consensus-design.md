# v0.9.2 — Team Onboarding + Consensus: Design Spec

**Date:** 2026-06-22
**Status:** Approved
**Version target:** v0.9.2
**Roadmap entry:** `project-docs/roadmap.md` — v0.9.2 Team Onboarding + Consensus

---

## Context

v0.9.2 is the first release targeting multi-human teams. The immediate use case is a monorepo with 4–5 members using different AI toolkits (Gemini + Codex baseline, Claude for some). All features are additive — no existing commands change behaviour.

The codebase is a single file: `synlynk/__init__.py` (~4,911 lines, stdlib only). All new code goes in this file. Tests go in `tests/test_synlynk.py` (369 passing). The SQLite database lives at `~/.synlynk/projects/<8-char-md5-of-git-root>/state.db`, shared across all worktrees of the same repo.

---

## Scope

Five features ship in v0.9.2:

1. **`synlynk join`** — new member onboards to an existing project
2. **`synlynk team status`** — team digest on demand
3. **Pull-before-write arbitration** — warn when remote has unpulled commits
4. **Token budgets on stories** — AI planner sets `estimated_tokens`; actual tracked from telemetry
5. **`synlynk decide`** — multi-agent consensus panel → signed Decision record

---

## Architecture: Shared Helpers

Three helper functions are extracted and shared between `init` (team mode), `join`, and `team status`. They live in `__init__.py` alongside the existing helpers.

### `_seed_devlog(username: str) -> None`

Creates `project-docs/devlogs/<username>.md` if it does not already exist. Writes a minimal header:

```markdown
# Devlog — @<username>

## <YYYY-MM-DD> — Joined project
Joined via `synlynk join`.
```

No-ops if the file already exists (idempotent).

### `_generate_ai_context_files(arch_context: str, git_summary: str) -> None`

Writes or updates CLAUDE.md, GEMINI.md, and AGENTS.md at the repo root. If a file already exists, appends a `## Context Snapshot (joined YYYY-MM-DD)` section at the end — it does **not** overwrite. If a file does not exist, creates it from the standard template (same as `init`).

`arch_context` is the output of `_run_scan()`. `git_summary` is the last 20 commit subjects from `git log --oneline -20`.

### `_build_team_digest() -> dict`

Reads from two sources — always both, no fallback:

- **SQLite `state.db`**: in-progress stories (status != `done`), `estimated_tokens`, `actual_tokens` per story, capability scores
- **`project-docs/devlogs/`**: last-active date per member (most recent `## YYYY-MM-DD` heading), shipped story count (count of `## ` headings as a proxy)

If `state.db` does not exist, skips the SQLite read and notes "No telemetry yet." If `project-docs/devlogs/` is empty, skips that read silently.

Returns:
```python
{
    "members": [{"user": str, "last_active": str, "stories_shipped": int}],
    "in_progress": [{"story_id": str, "title": str, "estimated_tokens": int|None, "actual_tokens": int}],
    "recently_completed": [{"story_id": str, "title": str, "estimated_tokens": int|None, "actual_tokens": int}],
    "top_todo": str | None,
}
```

`actual_tokens` for a story is summed from `telemetry.json` entries where `story_id` matches.

## Feature 1: `synlynk join`

### CLI

```
synlynk join
```

No required arguments. Username derived from `git config user.name`.

### Flow

1. Check `project-docs/` exists. If not: `error: project not initialized — run 'synlynk init' first` and exit 1.
2. Detect `username = git config user.name`.
3. Run `_run_scan()` to get `arch_context`.
4. Read `git log --oneline -20` to get `git_summary`.
5. Call `_generate_ai_context_files(arch_context, git_summary)`.
6. Call `_seed_devlog(username)`.
7. Set `mode = "team"` in `project-docs/.synlynk_config.json` if not already set.
8. Call `_check_upstream_divergence()` (arbitration guard, see Feature 3).
9. Call `_build_team_digest()` and print team digest.
10. Print recommended first task from `top_todo`.

### Terminal output

```
✓ Joined project as @nikhil
✓ Seeded devlog at project-docs/devlogs/nikhil.md
✓ Updated CLAUDE.md, GEMINI.md, AGENTS.md

TEAM (3 members)
  @alice    · last active 2026-06-21  · 4 stories shipped
  @bob      · last active 2026-06-20  · 2 stories shipped
  @nikhil   · joined now

IN PROGRESS
  story-a1b2  feat: v0.9.2 team onboarding   ~50K tokens est · 12K actual so far

RECOMMENDED FIRST TASK
  → Add synlynk join command (see todo.md)
```

## Feature 3: Pull-before-write Arbitration

### Helper: `_check_upstream_divergence() -> None`

```python
def _check_upstream_divergence() -> None:
    """Warn if remote has commits the local branch hasn't pulled."""
    try:
        local = subprocess.check_output(
            ["git", "rev-parse", "HEAD"], stderr=subprocess.DEVNULL
        ).strip()
        upstream = subprocess.check_output(
            ["git", "rev-parse", "@{u}"], stderr=subprocess.DEVNULL
        ).strip()
    except subprocess.CalledProcessError:
        return  # No upstream configured — solo repo, detached HEAD, CI. Silent no-op.
    if local != upstream:
        behind = subprocess.check_output(
            ["git", "rev-list", "--count", "HEAD..@{u}"], stderr=subprocess.DEVNULL
        ).strip().decode()
        print(f"⚠  Remote has {behind} commit(s) you haven't pulled. "
              f"Consider `git pull` before writing.\n   Continuing anyway...")
```

### Injection points

Called at the top of any function that writes to `project-docs/`:

- `checkpoint()` — before writing the checkpoint entry
- `update_costs()` — before appending to costs.md
- The devlog-write path in `exec_command()` — before updating the devlog
- `_seed_devlog()` — before creating the devlog file
- `cmd_decide()` with `--record` — before writing Decision files

Silently no-ops when no upstream is configured.

## Feature 4: Token Budgets on Stories

### Schema migration

`_get_db()` runs these on every connection (idempotent `ALTER TABLE` pattern already in use):

```sql
ALTER TABLE stories ADD COLUMN estimated_tokens INTEGER;
ALTER TABLE stories ADD COLUMN actual_tokens INTEGER;
```

Existing rows get `NULL` for both columns.

### CLI change

```
synlynk story create --title "Feat X" --engg backend --tokens 50000
```

`--tokens` is optional. When omitted, `estimated_tokens` stays `NULL`. The AI planner provides this flag — humans never need to.

### Actual token tracking

`actual_tokens` on a story is not stored directly in the `stories` table — it is computed at read time by summing `tokens_out + tokens_in` from `telemetry.json` entries where `story_id` matches. `cmd_story_list()` and `_build_team_digest()` both perform this computation.

### Display

`synlynk story list` gains two columns:

```
STORY ID    TITLE                         ENGG       EST TOK   ACTUAL TOK
story-a1b2  feat: v0.9.2 team onboarding  backend    50,000    12,341
story-c3d4  fix: scan path on Windows     platform   —         4,201
```

## Feature 5: `synlynk decide`

### CLI

```
synlynk decide "<topic>" --panel claude,agy,codex [--record]
```

- `<topic>` — positional string, quoted. Required.
- `--panel` — comma-separated list of agent names. Required. Each name must match a known exec target (`claude`, `agy`, `codex`) or a `.agents/<name>.json` config.
- `--record` — if present, writes the Decision record files. Without it, prints synthesis to terminal only (dry-run mode).

### Flow

1. Build panel prompt:
   ```
   You are part of a decision panel. Topic: "<topic>"
   Provide your analysis and recommendation in 200–400 words.
   State your position clearly in the final paragraph.
   ```
2. Dispatch each panel member sequentially via `dispatch_agent()` with the panel prompt. Collect stdout as each member's input. Timeout per member: 120 seconds.
3. Build synthesis prompt: concatenate all member inputs, then:
   ```
   The following are inputs from a decision panel on: "<topic>"

   <member inputs, labelled by name>

   Synthesize these into a single decision. In the final paragraph,
   state the decision clearly starting with "Decision:".
   ```
4. Dispatch the first panel member again with the synthesis prompt. Collect synthesis output.
5. Generate `decision_id = "dec-" + md5(f"{topic}{timestamp}")[:8]`.
6. Print synthesis to terminal.
7. If `--record`:
   - Call `_check_upstream_divergence()`.
   - Create `project-docs/decisions/` if absent.
   - Write `YYYY-MM-DD-<slug>.md` (human-readable).
   - Write `YYYY-MM-DD-<slug>.json` (machine-queryable).
   - Ed25519-sign the canonical JSON string using `_sign_capability_rating()` pattern (existing identity key at `~/.synlynk/identity.key`). Add `"signature": "<hex>"` field to the JSON.
   - Print: `✓ Decision recorded: project-docs/decisions/YYYY-MM-DD-<slug>.md`

`<slug>` is the topic lowercased, spaces replaced with hyphens, truncated to 40 chars.

### Decision record format

**`YYYY-MM-DD-<slug>.md`**:
```markdown
---
decision_id: dec-a1b2c3d4
topic: "Relay ownership model"
date: 2026-06-22
panel: [claude, agy, codex]
status: approved
---

## Topic
Relay ownership model for v0.9.3

## Panel Inputs

### claude
<verbatim stdout from claude dispatch>

### agy
<verbatim stdout from agy dispatch>

### codex
<verbatim stdout from codex dispatch>

## Synthesis
<verbatim synthesis output>

## Decision
<final paragraph extracted from synthesis>

> Signatures: see 2026-06-22-relay-ownership.json
```

**`YYYY-MM-DD-<slug>.json`**:
```json
{
  "decision_id": "dec-a1b2c3d4",
  "topic": "Relay ownership model",
  "date": "2026-06-22",
  "panel": ["claude", "agy", "codex"],
  "status": "approved",
  "inputs": {
    "claude": "<verbatim>",
    "agy": "<verbatim>",
    "codex": "<verbatim>"
  },
  "synthesis": "<verbatim>",
  "decision": "<final paragraph>",
  "signature": "ed25519:<hex>"
}
```

The `signature` field is the Ed25519 signature of the JSON string with keys sorted and `"signature"` field excluded.

## Error handling

| Scenario | Behaviour |
|---|---|
| `synlynk join` — `project-docs/` absent | Exit 1 with clear message |
| `synlynk join` — git not configured | Exit 1: "git config user.name not set" |
| `_check_upstream_divergence` — no upstream | Silent no-op |
| `synlynk decide` — panel member dispatch fails | Print warning, continue with remaining members. If all fail, exit 1. |
| `synlynk decide` — identity key absent | Print warning: "No identity key — decision will not be signed. Run `synlynk identity init` first." Write files without signature field. |
| `synlynk decide` — `project-docs/decisions/` absent | Create it automatically |
| `story create --tokens` — non-integer value | Exit 1 with usage error |
