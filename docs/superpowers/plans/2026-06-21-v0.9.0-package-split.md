# v0.9.0 Package Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split `bin/synlynk.py` (~4563 lines) into a `synlynk/` Python package with clean module boundaries, with no behavior changes. All 200+ existing tests must pass unchanged (except the import line).

**Architecture:** `bin/synlynk.py` becomes a 5-line shim. A new `synlynk/` package at repo root contains 13 modules. `synlynk/__init__.py` re-exports every public symbol so existing test imports work after changing one line. `install.sh` is updated to copy the `synlynk/` directory alongside the binary shim.

**Tech Stack:** Python 3 stdlib only, zero new dependencies. `pytest tests/test_synlynk.py -v` — all tests must pass after every task.

**IMPORTANT:** This is a pure refactor. Do NOT change any function logic during the split. Move code exactly as-is. If you notice a bug during the move, note it in a comment but do not fix it — that belongs in a separate commit.
